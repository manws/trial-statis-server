# 临床Levene方差齐性检验统计分析模块 (Clinical Levene Variance Homogeneity Test Statistics Module)

## 模块概述

临床Levene方差齐性检验统计分析模块提供全面的Levene方差齐性检验统计分析功能，用于临床数据中多组方差齐性的检验，是检验多组数据方差是否相等的经典统计方法。Levene检验在医学研究中广泛应用，如比较多个治疗组方差的齐性、验证方差分析的前提条件等。相比Bartlett检验，Levene检验对数据正态性假设不敏感，更加稳健。

## 模块功能

### 1. Levene统计量计算
- 计算多组数据方差齐性的Levene统计量
- 评估多组方差差异程度

### 2. 绝对偏差计算
- 计算各组数据相对于其中心的绝对偏差
- 为检验提供基础

### 3. 显著性检验
- 执行F检验判断方差是否齐性
- 提供不同显著性水平的判断

### 4. 方差分解
- 计算组间、组内和总平方和
- 分析变异来源

### 5. 统计解释
- 提供结果的临床意义解释
- 帮助理解统计结果的实际含义

## 临床应用价值

- **方差齐性检验**：验证多组数据方差是否齐性
- **统计方法选择**：为后续统计分析方法的选择提供依据
- **数据质量评估**：评估多组间变异程度的差异
- **假设检验验证**：验证方差分析等方法的前提条件

## 统计方法选择指南

### 1. Levene检验适用条件
- 各组数据相互独立
- 适用于2组及以上数据的方差比较
- 对数据正态性假设不敏感，比Bartlett检验更稳健
- 适用于偏态分布或含有异常值的数据

### 2. 临床应用场景
- 比较多个治疗组某项指标的变异性差异
- 验证方差分析前的方差齐性假设
- 检验不同处理组间变异程度的差异（特别是当数据不满足正态性时）

## 主要函数说明

### Levene方差齐性检验函数 - calculate_levene_test_variance_homogeneity

执行Levene方差齐性检验。在临床研究中，这用于检验多组数据的方差是否齐性，即比较多个组数据的变异程度是否存在显著差异。相比Bartlett检验，Levene检验对数据正态性假设不敏感，更加稳健。

**参数**：
- `data_list`: List[List[float]]，每个子列表代表一组数据

**返回值**：
- 包含Levene检验统计量和结果的字典

### 结果计算函数 - cal_result_hv_levene

生成Levene方差齐性检验统计分析的完整报告字典。此函数整合了Levene方差齐性检验的所有关键指标，生成标准化的字典格式报告，适用于临床研究报告的需求，提供全面的Levene检验结果。

**参数**：
- `data_list`: List[List[float]]，每个子列表代表一组数据

**返回值**：
- 包含Levene方差齐性检验统计分析指标的字典，包括：
  - `input_parameters`: 输入参数信息
  - `test_statistics`: 检验统计量
  - `significance_tests`: 显著性检验结果
  - `interpretation`: 统计解释

## 结果解读注意事项

1. **F统计量解释**：统计量越大表示方差差异越显著
2. **P值解释**：在零假设（各组方差相等）成立的情况下，观察到当前或更极端结果的概率
3. **显著性水平**：通常使用α=0.05作为判断方差齐性的标准
4. **临床意义**：方差齐性是方差分析等统计方法的重要前提条件

## 使用示例

```python
from app.stats.hv.hv_stats_levene import cal_result_hv_levene

# 创建示例数据，三组数据
data_list = [
    [10.2, 12.1, 9.8, 11.5, 13.2],  # 第一组数据
    [8.7, 9.2, 8.9, 9.5, 10.1],     # 第二组数据
    [11.3, 10.8, 12.5, 11.9, 10.6]  # 第三组数据
]

# 计算Levene方差齐性检验
result = cal_result_hv_levene(data_list)

# 查看结果
print(f"输入参数: {result['input_parameters']}")
print(f"检验统计量: {result['test_statistics']}")
print(f"显著性检验: {result['significance_tests']}")
print(f"统计解释: {result['interpretation']}")
```
