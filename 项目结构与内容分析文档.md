# Trial Statistics Server 项目结构与内容分析文档

> **项目名称**: Trial Statistics Server（临床试验统计服务）  
> **技术栈**: Python + FastAPI + aiomysql + Redis  
> **版本**: 1.0.0  
> **文档生成日期**: 2026-04-02

---

## 一、项目概述

Trial Statistics Server 是一个基于 **FastAPI** 构建的临床试验统计分析后端服务，提供 RESTful API 接口，涵盖 **13 大类、约 60 个核心统计算法**，为临床研究数据分析提供标准化的计算服务。

### 核心能力

| 能力 | 说明 |
|------|------|
| 统计分析 | 覆盖参数检验、非参数检验、生存分析、回归分析等主流医学统计方法 |
| 异步高性能 | 基于 FastAPI + uvicorn 提供异步 API 服务 |
| 数据持久化 | 支持 MySQL（异步连接池）和 Redis 缓存 |
| 生命周期管理 | 完善的应用启动/关闭资源管理和跨平台信号处理 |

## 三、技术架构详解

### 3.1 应用入口 (`main.py`)

- 使用 FastAPI 创建应用实例，配置 `lifespan` 生命周期管理
- 启动时通过 `AppInitializer` 初始化 Redis 和 MySQL 连接池
- 关闭时自动清理资源，支持跨平台信号处理（Windows SIGINT/SIGBREAK, Unix SIGTERM/SIGINT）
- 开发环境默认监听 `0.0.0.0:8000`，启用热重载

### 3.2 路由架构

API 路由采用三级嵌套结构：

```
/api/v1                        ← 顶层路由（controllers/__init__.py）
  └── /stats                   ← 统计模块路由（controllers/stats/__init__.py）
       ├── /base/*             ← 基础统计（13 个接口）
       ├── /anova/*            ← 方差分析
       ├── /chi/*              ← 卡方检验
       ├── /t/*                ← T 检验
       ├── /z/*                ← Z 检验
       ├── /rs/*               ← 秩和检验
       ├── /hv/*               ← 方差齐性检验
       ├── /sur/*              ← 生存分析
       ├── /cor/*              ← 相关分析
       ├── /reg/*              ← 回归分析
       ├── /clu/*              ← 聚类分析
       ├── /cq/*               ← Cochran Q 检验
       └── /runs/*             ← 游程检验
```

### 3.3 统一响应格式

所有接口返回统一的 `ResultData` 模型：

```python
class ResultData(BaseModel):
    code: int          # 状态码（200 成功，500 失败）
    message: str       # 描述信息
    result: Dict       # 计算结果（含 report 和可选 data 字段）
```

通过 `_wrap()` 工具函数统一封装，保证响应结构一致性。

### 3.4 核心基础设施 (`core/`)

| 模块 | 功能 | 关键特性 |
|------|------|----------|
| `app_init.py` | 应用初始化管理器 | 管理所有组件的初始化与清理，防止重复初始化，按逆序清理 |
| `mysql_pool.py` | MySQL 异步连接池 | 基于 aiomysql，双重检查锁单例，支持事务管理、上下文管理器 |
| `redis_pool.py` | Redis 异步连接池 | 支持密码认证、健康检查、自动重连、超时配置 |
| `sql_executor.py` | SQL 执行器 | 封装 get_one/get_value/get_list/execute_sql 四种常用操作 |

### 3.5 数据模型 (`schemas/`)

**通用数据结构：**

```python
class StatsData(BaseModel):
    field_name: str           # 字段名称
    data_list: List[float]    # 数据列表

class StatsName(BaseModel):
    field_name: str           # 字段名称
    name_list: List[str]      # 名称列表
```

每个统计模块都有独立的请求参数模型（如 `BaseParamBino1`、`TParamIndep` 等），通过 Pydantic 进行参数校验。

### 3.6 日志系统 (`utils/LoggerHelper.py`)

- 支持文件日志 + 控制台日志双输出
- 日志文件按日期命名（`yyyy-MM-dd.log`），支持滚动（10MB/文件，保留 5 个备份）
- 提供 `debug/info/warning/error/critical/exception` 全级别日志
- 类方法 + 模块级便捷函数双重访问方式

## 六、API 接口设计模式

### 请求流程

```
客户端请求
  → FastAPI 路由匹配（/api/v1/stats/{module}/{method}）
    → Pydantic 参数校验
      → Controller 调用 stats 算法函数
        → 算法计算返回 report 对象
          → _wrap() 统一封装为 ResultData
            → JSON 响应返回客户端
```

### 异常处理策略

- **ValueError**: 返回 HTTP 400（参数校验失败）
- **其他异常**: 记录日志，返回 HTTP 500（含错误信息）

## 九、数据模型一致性检查报告（原始版本，保留便于回溯）

通过逐一比对 `schemas/request_data/` 中的 Pydantic 模型定义与 `controllers/stats/` 中的实际调用，发现以下不一致问题：

### 9.1 严重问题（运行时必报错）

#### 问题 1：`BaseParamBino2` 字段名与控制器调用不匹配

- **模型定义** (`base_param.py:28-34`):
  ```python
  class BaseParamBino2(BaseModel):
      confidence_level: float   # 字段名是 confidence_level
      sample_size: int
      sample_posi_num: int      # 字段名是 sample_posi_num
  ```
- **控制器调用** (`base_controller.py:185`):
  ```python
  result = cal_result_bino2(param.posi_num, param.sample_size, param.confidence)
  ```
- **问题**: 控制器访问 `param.posi_num` 和 `param.confidence`，但模型中定义的是 `sample_posi_num` 和 `confidence_level`，会触发 `AttributeError`。

#### 问题 2：`BaseParamBino3` 字段名与控制器调用不匹配

- **模型定义** (`base_param.py:37-43`):
  ```python
  class BaseParamBino3(BaseModel):
      total_posi_p: float
      sample_size: int
      sample_posi_num: int      # 字段名是 sample_posi_num
  ```
- **控制器调用** (`base_controller.py:208`):
  ```python
  result = cal_result_bino3(param.sample_size, param.posi_num, param.total_posi_p)
  ```
- **问题**: 控制器访问 `param.posi_num`，但模型中定义的是 `sample_posi_num`，会触发 `AttributeError`。

#### 问题 3：`BaseParamBino4` 字段名与控制器调用不匹配

- **模型定义** (`base_param.py:46-54`):
  ```python
  class BaseParamBino4(BaseModel):
      sample1_num: int
      sample1_posi_num: int
      sample2_num: int
      sample2_posi_num: int
  ```
- **控制器调用** (`base_controller.py:231`):
  ```python
  result = cal_result_bino4(param.posi_num1, param.sample_size1, param.posi_num2, param.sample_size2)
  ```
- **问题**: 控制器使用 `posi_num1/sample_size1/posi_num2/sample_size2`，但模型定义的是 `sample1_posi_num/sample1_num/sample2_posi_num/sample2_num`，全部不匹配。

#### 问题 4：`BaseParamPois1` 字段名与控制器调用不匹配

- **模型定义** (`base_param.py:57-63`):
  ```python
  class BaseParamPois1(BaseModel):
      total_avg: float
      total_p: float
  ```
